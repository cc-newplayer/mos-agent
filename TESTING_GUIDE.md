# 闭环测试使用指南

## ⚠️ 重要提示

每次运行测试脚本都会：
1. **创建新的对话**
2. **消耗系统积分**
3. **增加服务器压力**
4. **可能导致其他闭环排队**

**只在明确需要时运行测试，不要频繁运行。**

---

## 测试单个用例

```bash
python test_single_case.py <case_id>
```

例如，测试 Case 22（KOL 跟评闭环）：
```bash
python test_single_case.py 22
```

**流程：**
1. 创建一个新对话
2. 输入指定的指令
3. 轮询等待报告生成（最多 10 分钟）
4. 保存追踪日志

---

## 测试多个闭环用例

```bash
python test_kol_loop.py
```

**流程：**
1. 测试所有闭环用例（Cases 20-24）
2. 每个用例创建一个新对话
3. 总共创建 5 个对话
4. 生成测试报告
5. 推送到飞书

---

## 测试所有决策对话用例

```bash
python test_kol_complete.py
```

**流程：**
1. 测试所有决策对话用例（Cases 19-26）
2. 每个用例创建一个新对话
3. 总共创建 8 个对话
4. 生成测试报告
5. 推送到飞书

---

## 架构说明

### 配置驱动的路由

每个用例在加载时会自动识别其类型和配置：

```python
{
    "case_id": 22,
    "description": "事件源触发KOL跟评闭环",
    "module": "决策对话",
    "type": "closedloop",  # 用例类型
    "instruction": "用一个引流号对elonmusk最新的一篇帖子执行KOL跟评闭环"  # 指令
}
```

### 执行流程

1. **加载用例** → 识别类型和配置
2. **路由** → 根据类型调用相应处理器
3. **执行** → 闭环处理器执行测试
4. **轮询** → 等待报告生成
5. **保存** → 保存追踪日志和截图

---

## 注意事项

1. **不要频繁运行测试** - 每次都会消耗积分
2. **等待报告完成** - 轮询最多 10 分钟
3. **检查追踪日志** - 在 `traces/` 目录中
4. **查看截图** - 在 `screenshots/` 目录中
